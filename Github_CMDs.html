Summary of Common Git Commands

This cheat sheet covers basic Git commands such as setting up repositories, making commits, branching, working with remote repositories, and undoing changes. Whether you are just starting with Git or looking to refresh your skills, these commands will help you manage your Git workflows efficiently.

1. Setup Commands

git config --global user.name "Your Name" – Set the username for all repositories on your system.

git config --global user.email "you@example.com" – Set the email address for all repositories.

Example: If your name is Mohan and your email is mohan@example.com:

git config --global user.name "Mohan"

git config --global user.email "mohan@example.com"

2. Creating Repositories

git init – Initialize a new Git repository in the current directory.

git clone [url] – Clone an existing repository from the provided URL.

Example: To initialize a new repo:

git init

Example: To clone a repository:

git clone https://github.com/user/repository.git

3. Basic Snapshotting

git add [file] – Stage a specific file for commit.

git add . – Stage all changes in the current directory for commit.

git commit -m "message" – Commit the staged changes with a message.

git status – Check the status of your working directory and staged changes.

git log – View the history of commits in the current branch.

Example: To stage all changes and commit them with a message:

git add .

git commit -m "Added new features"

4. Branching and Merging

git branch – List all branches (current branch is highlighted).

git branch [branch-name] – Create a new branch with the specified name.

git checkout [branch-name] – Switch to the specified branch.

git merge [branch-name] – Merge the specified branch into the current branch.

git branch -d [branch-name] – Delete the specified branch.

Example: To create and switch to a new branch called "feature-branch":

git branch feature-branch

git checkout feature-branch

Example: To merge "feature-branch" into the main branch:

git checkout main

git merge feature-branch

5. Remote Repositories

git remote -v – List all configured remote repositories.

git remote add origin [url] – Add a remote repository with the name "origin".

git push origin [branch] – Push the specified branch to the remote repository.

git pull – Fetch and merge changes from the remote repository into the current branch.

Example: To add a remote repository and push changes:

git remote add origin https://github.com/user/repository.git

git push origin main

6. Undoing Changes

git checkout -- [file] – Discard changes in the working directory to the specified file.

git reset [file] – Unstage a file that was added to the staging area.

git revert [commit] – Revert a specific commit by creating a new commit that undoes it.

git reset --hard [commit] – Reset the working directory and staging area to a specific commit (discarding changes).

Example: To undo changes to a file:

git checkout -- file.txt

7. Stashing Changes

git stash – Save uncommitted changes for later and revert the working directory to the last commit.

git stash pop – Reapply the last stashed changes and remove them from the stash list.

git stash list – List all stashed changes.

Example: To stash changes and later reapply them:

git stash

git stash pop
